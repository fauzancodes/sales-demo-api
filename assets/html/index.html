Sales Demo API Documentation

Welcome to the Sales Demo API documentation. Below are the features of the API:

  • Login and register endpoints supported with jwt authentication
  • The password field is protected by bcrypt hashing
  • Endpoint for email verification, forgot password, getting logged in user data, updating user data, and deleting user accounts
  • The email verification endpoint is equipped with the function to send an email to the user and check the verification back
  • The endpoint for forgotten passwords is equipped with a function to send an email to the user in the form of a verification token which is then used to create a new password
  • CRUD endpoints for product categories, products themselves, product stock, and product image uploads
  • Product stock is supported by stock change history
  • CRUD endpoints for customers
  • CRUD endpoints for sales and details
  • Sales endpoints are equipped with validation of subtotal calculations, discounts, taxes, and totals to be paid
  • Sales endpoints are equipped with product stock validation and product stock reduction
  • Sales endpoints are equipped with endpoints to send sales invoices to customer emails
  • All GET endpoints are equipped with dynamic pagination, filter, search, limit, offset, order, and sort
  • All POST endpoints are equipped with validation for each field
  • All endpoints are protected with query param sanitization to prevent SQL injection
